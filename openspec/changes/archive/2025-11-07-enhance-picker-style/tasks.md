# Tasks: 签名选择对话框样式优化

## 任务列表

### Task 1: 卡片视觉增强

**目标**：提升卡片的视觉层次感和美观度

**工作内容**：
- 更新 `.signature-card` 的 box-shadow（正常 0 1px 3px，hover 0 4px 12px）
- 添加 hover 时的 translateY(-1px) 效果
- 优化 selected 状态的阴影和边框（添加蓝色光晕）

**验证**：
- hover 时卡片有轻微上升和阴影增强
- selected 时显示 2px 蓝色边框和淡蓝色光晕
- 过渡动画平滑

**依赖**：无

---

### Task 2: 内容区域优化

**目标**：改进搜索区域、名称和介绍的布局，支持横向滚动，优化间距

**工作内容**：
- 搜索区域 padding 从 `q-pa-md` 改为 `q-pa-sm`
- 描述文本下方 margin 从 `q-mb-md` 改为 `q-mb-sm`
- 搜索输入框下方 margin 从 `q-mb-md` 改为 `q-mb-sm`
- 卡片间距从 `q-mb-sm` 改为 `q-mb-xs`
- 移除主内容区域的 `padding-bottom: 100px`，改为自适应布局
- 名称和介绍各自添加横向滚动容器类
- 名称：单行，支持水平滚动，不换行
- 介绍：最多 2 行，超出支持水平滚动
- 图片区间距为 8px（margin: 0 8px）

**验证**：
- 搜索区域更紧凑
- 卡片间距更紧凑
- 长名称可水平滚动而不换行
- 长介绍限制 2 行且可水平滚动
- 整体布局更美观紧凑

**依赖**：Task 1（视觉效果的基础）

---

### Task 3: 选择交互灵活性

**目标**：支持点击已选项取消，增加选中图标的视觉效果

**工作内容**：
- 修改 `selectSignature()` 逻辑：若点击的是已选项，则取消选择
- 为选中图标添加 CSS 动画类 `selection-glow`
- 实现渐显透明光晕效果（radial-gradient，opacity 从 0.8 → 0.4 → 0.8）
- 光晕动画：scale 从 1 → 1.1 → 1，周期 1.5s
- 光晕采用 `radial-gradient(circle, rgba(33,150,243,0.3), transparent)`

**验证**：
- 点击未选项时选中
- 点击已选项时取消
- 选中时有光晕动画效果，脉冲流畅

**依赖**：Task 1

---

### Task 4: 底部操作栏固定

**目标**：使底部按钮固定在对话框底部，支持毛玻璃背景

**工作内容**：
- 将 `<q-card-actions>` 改为 `position: sticky; bottom: 0`
- 添加毛玻璃背景（backdrop-filter: blur(10px); background: rgba(255,255,255,0.1)）
- 添加上边界线（border-top: 1px solid rgba(0,0,0,0.08)）
- 设置 z-index: 10 确保始终在内容之上
- 不需要添加 padding-bottom 预留空间（因为已优化了内容区域间距）

**验证**：
- 滚动内容时，按钮始终可见
- 按钮背景显示毛玻璃效果
- 无遮挡内容的情况
- 边界线清晰可见

**依赖**：Task 1

---

### Task 5: 隐藏主对话框滚动条

**目标**：设计更精致的对话框，隐藏主滚动条，改善视觉体验

**工作内容**：
- 应用 `[&::-webkit-scrollbar]:hidden` 到对话框卡片元素
- 使用 Tailwind CSS 任意值语法
- 保留 `[overflow:overlay]` 确保滚动正常

**验证**：
- 对话框主滚动条被隐藏
- 用户仍可通过鼠标滚轮/触控板滚动
- 内容卡片内的横向滚动条仍可见
- 对话框布局更清爽

**依赖**：无（独立改进）

---

### Task 6: 功能验收与兼容性测试

**目标**：确保所有改进协同工作，无破坏性改动

**工作内容**：
- 完整流程测试：打开对话框 → 搜索 → 选择 → 取消 → 再选 → 确认
- 视觉验证：所有样式效果在不同屏幕尺寸上正常显示
- 浏览器兼容性：Chrome、Firefox、Safari、Edge
- 性能检查：是否有滚动卡顿或动画掉帧
- 确认不破坏导出流程的其他对话框

**验证**：
- 所有 6 个测试场景通过
- 未发现新的 bug
- 代码 lint 通过

**依赖**：Task 1-5
