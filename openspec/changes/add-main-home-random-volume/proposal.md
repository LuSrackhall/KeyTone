# Change: 在主页面相关设置末尾新增随机音量与按下抬起随机单独控制

## Why

当前主页面虽然支持全局/分离/按下抬起等多层音量控制，但都属于“确定性音量”。
希望获得“每次触发略有不同”的用户，必须去修改专辑内容或手工制作随机键音，门槛较高。

需要提供一个对所有已载入键音专辑都可一键生效的“随机音量”能力，且必须满足两点：
- 每次播放都不超过当前实际音量（只做衰减，不做放大）；
- 衰减范围受控，不会轻易进入近静音体验。

## What Changes

- 在“设置页面 → 主页面相关设置项”末尾新增“随机音量”能力：
  - 新增总开关（默认关闭）。
  - 新增“随机降幅上限”数值输入项（默认 `3`，仅开关开启时展示）。
- 新增“按下/抬起随机音量单独控制”能力：
  - 新增总开关（默认关闭）。
  - 开启后展示独立展开项，按“全局 → 键盘 → 鼠标”顺序提供六个事件态节点：
    - 全局按下、全局抬起、键盘按下、键盘抬起、鼠标按下、鼠标抬起。
  - 每个节点包含“启用开关 + 随机降幅上限输入框”（输入框仅该节点启用时展示，默认 `3`）。
- 文案命名优化（zh-CN/en-US）：
  - “随机音量(全局)” 与 “随机降幅上限(全局)”；
  - “随机音量(按下/抬起单独控制)”；
  - 单独控制说明明确“若开启全局随机音量设置，生效过程将会与全局随机音量设置叠加”；
  - 节点输入统一采用“随机降幅上限(场景)”格式（如“随机降幅上限(全局按下)”).
- 随机算法采用“在当前 volume 值上随机减去一个值”：
  - 每次播放时生成随机降幅 `d ∈ [0, maxReduceRatio]`；
  - 叠加随机层 `deltaVolume = -d`；
  - 因 `deltaVolume <= 0`，故随机层只会衰减，不会放大。
- 安全边界：
  - `maxReduceRatio` 仅限制为非负数（`>= 0`），不设置上限，允许用户配置更大的随机降幅；
  - 保证随机音量不会超过当前实际音量；当用户配置极大降幅时，允许出现接近静音的随机结果。
- 播放链路扩展：
  - 全局音量处理 → 分离设备音量 → 按下/抬起音量 → 全局随机音量层（若开启）→ 按下/抬起随机音量层（若开启且节点启用）。
  - 当全局随机与按下/抬起随机同时开启时，随机降幅允许叠加两次。
- 持久化与迁移：
  - 新增 `main_home.random_volume_processing.*` 配置键；
  - 对旧配置做增量补齐，不影响既有键与行为。

## Impact

- Affected specs:
  - `playback-album-routing`（新增随机音量 requirement）
- Affected code (planned):
  - `frontend/src/pages/SettingPageChildren/MainHome_setting.vue`
  - `frontend/src/stores/setting-store.ts`
  - `frontend/src/i18n/zh-CN/index.json`
  - `frontend/src/i18n/en-US/index.json`
  - `sdk/config/config.go`
  - `sdk/keySound/keySound.go`
- Non-goals:
  - 不新增专辑级或单键级随机音量规则；
  - 不引入多种随机策略模式切换（先提供单一稳定策略）。
